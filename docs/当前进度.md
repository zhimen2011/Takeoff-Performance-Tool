建议下次接续时优先从以下状态开始：

4. 检查最新输出目录中：
   - `STASOUT.out` 是原始 STAS 输出。

## 当前已完成的关键功能

- DearPyGui 桌面界面已作为当前主界面。
- 跑道复选框支持全选、全不选、反选。
- 队列方案会保存温度、风速、QNH、QNHREF 输出标注和每条顺序项条件。
- 队列方案不保存机型、机场、跑道和手册起飞分析格式。
- 临时起飞分析格式仅在未选择手册格式时生成；选择手册起飞分析格式后不再同时生成临时格式。
- 手册报告保留 `      ELEVATION` 前导空格，并按段分页。
- 手册报告会高亮 `ANTI-ICE ENG ONLY` 和 `10% DERATE`，不高亮 `20% DERATE`。
- 队列临时 Word/PDF 已移除 `SCENARIO / Aircraft / Raw output / Section` 等调试表头。
- `APTRWY_MASTER.RWY` 作为机场主库，`APTRWY.RWY` 作为 STAS 实际读取的运行文件。
- GUI 已支持外部 `.RWY` / `.rwy` / `.stx` 文件导入到 `APTRWY_MASTER.RWY`，支持预览、跳过、覆盖、备份和导入后刷新列表。
- AIRPORT2 `#INT` 交叉口短跑道已在 UI/校验/运行文件中自动生成，主库仍保留外部机场原始块。
- AIRPORT2 单引号文本行已清理尾部航空日期，连续 `H ` EOSID 行已合并。
- STAS 报告导出前会从运行用 `APTRWY.RWY` 补全超长特殊程序文本，生成 `STASOUT.enriched.out`。
- Word、PDF、队列报告优先读取 `report_output_path`；没有补全文件时回退到原始 `STASOUT.out`。
- `NO EMERGENCY TURN` 明确不补充，保持现有跳过/保留逻辑。

## 2026-05-16 任务队列与顺序存档更新

本轮已把“全局开关笛卡尔积”的思路改为“Scenario 队列驱动”：

- GUI 增加“输出顺序队列”，用户可以按需要逐条加入 Scenario。
- 每条 Scenario 可独立设置道面条件、污染深度、推力、引气/空调、防冰。
- 用户可以通过上移、下移、删除、清空队列控制输出顺序。
- 保留“旧模板4项”按钮，用于快速生成原模板默认顺序。
- 删除旧的“批量组合生成”“组引气顺序”“当前减推力”入口。
- 新增“队列方案（不含机型/机场/跑道/报告格式）”，可保存、加载、删除用户自定义队列。
- 队列方案保存 `temperature_range`、`wind_range`、`qnh_ref`、`describe_qnh_ref`、`runway_condition`、`contamination_depth`、`thrust_option`、`derate`、`bleed`、`anti_icing`。
- `derate` 字段保留用于兼容旧顺序存档；GUI 新增队列项使用 `thrust_option`。
- 机型、机场、跑道和报告格式不进入队列方案，因此同一计算条件队列可以换任意机型、机场/跑道和报告格式使用。

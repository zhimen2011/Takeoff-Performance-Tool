# 手册起飞分析版式维护

本文档说明后续如何调整“手册起飞分析格式”的版式，避免每次修改都重新扫描代码。

## 1. 当前生成方式

手册报告由以下模块生成：

- 模板目录：`templates/reports/manual_takeoff/`
- 模板注册表：`templates/reports/manual_takeoff/templates.toml`
- 导出代码：`src/stas_app/exporters/manual_takeoff_report.py`
- 队列手册合并入口：`src/stas_app/exporters/queue_report.py`

当前逻辑是：

```text
读取 Word 模板
  -> 保留模板的页眉、页脚、页面设置
  -> 清空模板正文
  -> 按 STAS 输出重新写入正文
  -> 保存为手册起飞分析 Word
  -> 如启用 PDF，再由 Word COM 转 PDF
```

所以，模板正文内容不会被保留。模板主要负责页面和页眉页脚，正文由代码重新写入。

## 2. 可以直接改 Word 模板的内容

以下版式主要来自 `.docx` 模板，可以直接修改模板文件：

- 页眉。
- 页脚。
- 页边距。
- 纸张大小。
- 页面方向。
- 页码样式。
- 页眉里的 logo、标题、表格线。
- Word 文档的 section 页面设置。

当前模板文件在：

```text
templates/reports/manual_takeoff/738_normal.docx
templates/reports/manual_takeoff/777f_normal.docx
templates/reports/manual_takeoff/777f_derate.docx
templates/reports/manual_takeoff/777f_bump.docx
```

如果只是替换现有版式：

1. 用 Word 打开对应 `.docx`。
2. 修改页眉、页边距、纸张或页面方向。
3. 保持文件名不变。
4. 保存后重新运行程序导出报告。

如果要新增一个手册格式选项：

1. 把新的 `.docx` 放入 `templates/reports/manual_takeoff/`。
2. 修改 `templates/reports/manual_takeoff/templates.toml`。
3. 增加一段 `[[templates]]`，填写：

```toml
[[templates]]
id = "new_template_id"
label = "界面显示名称"
filename = "new_template.docx"
aircraft_codes = ["738"]
thrust_options = ["", "正常", "NORMAL"]
```

字段说明：

- `id`：内部唯一编号，不能和已有模板重复。
- `label`：界面下拉框显示的名称。
- `filename`：模板文件名。
- `aircraft_codes`：允许使用该模板的机型。
- `thrust_options`：允许使用该模板的推力选项。

## 3. 需要改代码的正文版式

以下内容目前由 `manual_takeoff_report.py` 控制，直接改 Word 模板正文不会生效：

- 正文字体。
- 正文字号。
- 正文行距。
- `ELEVATION` 前导空格。
- 每个 `ELEVATION` 段是否分页。
- 长段落是否自动压缩字号和行距。
- 正文每行如何写入 Word。
- STAS 输出如何拆分成多个手册页面。

当前关键位置：

```text
src/stas_app/exporters/manual_takeoff_report.py
```

核心函数：

- `split_manual_takeoff_report_sections()`：决定 STAS 输出如何拆成手册段落。
- `manual_section_layout()`：决定某一段使用正常、紧凑还是更紧凑排版。
- `ManualTakeoffReportExporter._generate()`：真正向 Word 写入正文。

## 4. 当前正文排版参数

当前使用三档排版：

```text
正常：
  字号 7.5 pt
  行距 9.3 pt

紧凑：
  字号 7.2 pt
  行距 8.4 pt

更紧凑：
  字号 6.8 pt
  行距 7.4 pt
```

当前估算规则：

```text
每行估算宽度：108 字符
估算行数 >= 62：使用紧凑排版
估算行数 >= 70：使用更紧凑排版
```

对应常量在 `manual_takeoff_report.py` 顶部：

```text
MANUAL_SECTION_NORMAL_LAYOUT
MANUAL_SECTION_COMPACT_LAYOUT
MANUAL_SECTION_TIGHT_LAYOUT
MANUAL_SECTION_CHARACTERS_PER_LINE
MANUAL_SECTION_COMPACT_LINE_THRESHOLD
MANUAL_SECTION_TIGHT_LINE_THRESHOLD
```

如果需要进一步压缩页面，优先调整：

1. `MANUAL_SECTION_TIGHT_LAYOUT.line_spacing_pt`
2. `MANUAL_SECTION_TIGHT_LAYOUT.font_size_pt`
3. `MANUAL_SECTION_TIGHT_LINE_THRESHOLD`
4. `MANUAL_SECTION_COMPACT_LINE_THRESHOLD`

## 5. ELEVATION 缩进规则

当前 `ELEVATION` 行前会保留 6 个空格：

```text
      ELEVATION
```

这个缩进来自 `src/stas_app/exporters/word_report.py` 的分段逻辑：

```text
split_stas_report_sections()
```

手册导出层不再清除这个前导空格。相关测试在：

```text
tests/test_manual_takeoff_report_exporter.py
```

如果后续想改成 7 个空格、4 个空格或顶格，需要同时修改：

1. `src/stas_app/exporters/word_report.py`
2. `tests/test_manual_takeoff_report_exporter.py`
3. 如影响临时格式，还要检查 `tests/test_word_report_exporter.py`

## 6. 分页规则

当前规则：

- 每个 `ELEVATION` 段作为一个独立页面。
- 第一段不强制前置分页。
- 第二段开始，在该段第一个段落上设置 `page_break_before = True`。
- 不再额外插入一个只包含分页符的空段落，减少空白页风险。
- 同一段内部使用 `keep_together` 和 `keep_with_next`，尽量保持整段不被 Word 拆开。

注意：Word 的最终分页由 Word 渲染引擎决定。代码能降低跨页风险，但如果某个 `ELEVATION` 段内容本身超过一页容量，仍需要进一步压缩字号、行距或调整模板页边距。

## 7. 正文关键词高亮

当前手册格式正文会对以下关键词使用 Word 原生黄色文本高亮：

```text
ANTI-ICE ENG ONLY
10% DERATE
```

不会高亮：

```text
20% DERATE
```

高亮逻辑在：

```text
src/stas_app/exporters/manual_takeoff_report.py
```

相关常量：

```text
MANUAL_HIGHLIGHT_TERMS
```

实现方式：

- 导出时保持原始文本不变。
- 将命中的关键词拆成单独 Word run。
- 只给关键词 run 设置黄色高亮。
- 普通文本 run 不加高亮。

PDF 由 Word COM 从 `.docx` 转换生成，正常情况下 Word 文本高亮会保留到 PDF。最终仍以本机 Microsoft Word 的导出效果为准。

如果以后要增减高亮词：

1. 修改 `MANUAL_HIGHLIGHT_TERMS`。
2. 更新 `tests/test_manual_takeoff_report_exporter.py`。
3. 运行手册导出测试。

## 8. 常见修改场景

### 8.1 只改页眉或页边距

操作：

1. 打开对应模板 `.docx`。
2. 修改页眉、页边距、纸张或页面方向。
3. 保存模板。
4. 用程序重新生成报告检查效果。

不需要改代码。

### 8.2 改正文行距或字号

操作：

1. 修改 `src/stas_app/exporters/manual_takeoff_report.py` 中的三档排版常量。
2. 更新 `tests/test_manual_takeoff_report_exporter.py`。
3. 运行测试。

建议测试命令：

```powershell
python -m pytest tests/test_manual_takeoff_report_exporter.py
python -m pytest
```

### 8.3 改每段是否独立分页

操作：

1. 修改 `ManualTakeoffReportExporter._generate()`。
2. 重点检查 `paragraph_format.page_break_before`。
3. 更新 `tests/test_manual_takeoff_report_exporter.py`。
4. 用真实 Word 打开导出文件确认页数。

### 8.4 改 STAS 输出正文内容

如果想删掉某些 STAS 输出行、调整正文行顺序、增加固定标题，需要修改：

```text
split_manual_takeoff_report_sections()
ManualTakeoffReportExporter._generate()
```

如果临时格式和手册格式都要改变，还要检查：

```text
src/stas_app/exporters/word_report.py
```

### 8.5 改正文高亮规则

操作：

1. 修改 `MANUAL_HIGHLIGHT_TERMS`。
2. 不要在高亮逻辑里改动原始文本。
3. 更新 `tests/test_manual_takeoff_report_exporter.py` 的高亮断言。
4. 导出 Word 后检查关键词是否只高亮目标文本。

## 9. 修改后的验证清单

建议每次修改手册版式后检查：

- 单条计算是否能生成手册 Word。
- 队列计算是否能生成队列手册 Word。
- `ELEVATION` 缩进是否符合样例。
- 每个 `ELEVATION` 段是否从新页开始。
- 是否出现整页空白。
- 长页面是否仍跨页。
- `ANTI-ICE ENG ONLY` 和 `10% DERATE` 是否高亮。
- `20% DERATE` 是否未高亮。
- PDF 转换是否正常。

建议命令：

```powershell
python -m pytest tests/test_manual_takeoff_report_exporter.py
python -m pytest tests/test_queue_report_exporter.py
python -m pytest
```

如果只是改 `.docx` 模板，自动化测试只能检查代码没有损坏，最终视觉效果仍需要用 Word 打开实际导出的报告确认。
# 临时起飞分析模板补充

临时起飞分析模板也放在 `templates/reports/manual_takeoff/`，但不使用手册模板注册表：

```text
templates/reports/manual_takeoff/temporary_default.docx
templates/reports/manual_takeoff/temporary_templates.toml
```

`temporary_default.docx` 负责临时起飞分析的页眉、logo、页面设置和正文样式；`WordReportExporter` 会清空模板正文，再写入 STAS 输出内容。后续只调整临时起飞分析样式时，优先修改 `temporary_default.docx`。
# 报告日期覆盖说明

手册起飞分析和临时起飞分析都支持在导出 Word 前替换正文中的 `DATED DD-MMM-YYYY`。该逻辑不写回 `STASOUT.out`，只影响 Word/PDF。

相关代码：

```text
src/stas_app/utils/aviation_date.py
src/stas_app/exporters/word_report.py
src/stas_app/exporters/manual_takeoff_report.py
```

如果后续调整日期显示格式，应优先修改 `aviation_date.py`，不要在多个导出器里分别拼接日期字符串。
